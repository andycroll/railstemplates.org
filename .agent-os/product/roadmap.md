# Product Roadmap

> Last Updated: 2025-08-13
> Version: 1.0.0
> Status: Planning

## Phase 1: Foundation

**Goal:** Launch minimal viable template repository with core functionality
**Success Criteria:** 10+ working templates, successful template application, basic search

### Features

- [ ] Static site with Jekyll setup - Basic Jekyll site structure `S`
- [ ] Template catalog page - Display all available templates `M`
- [ ] Individual template pages - Detailed view for each template `S`
- [ ] Copy-to-clipboard for URLs - Easy template URL copying `XS`
- [ ] Basic categorization - Group templates by type `S`
- [ ] Search functionality - Find templates by keyword `S`
- [ ] Rails version compatibility display - Show supported versions `XS`

### Dependencies

- Domain setup (railstemplates.org)
- GitHub repository creation
- Initial template collection

## Phase 2: Enhanced Discovery

**Goal:** Improve template discovery and documentation
**Success Criteria:** 25+ templates, compatibility matrix, contribution guide

### Features

- [ ] Compatibility matrix - Show which templates work together `M`
- [ ] Template combination guides - Document common template sets `M`
- [ ] Contributing guide - How to submit new templates `S`
- [ ] Template testing CI - Automated validation of templates `L`
- [ ] RSS/Atom feed - Subscribe to new templates `S`
- [ ] Template versioning - Track template changes over time `M`

### Dependencies

- Test Rails application setup
- GitHub Actions configuration
- Community feedback from Phase 1

## Phase 3: Community & Advanced Features

**Goal:** Build community features and advanced functionality
**Success Criteria:** Active community contributions, 50+ templates, advanced search

### Features

- [ ] Template ratings/reviews - Community feedback system `L`
- [ ] User submissions via PR - Streamlined contribution process `M`
- [ ] Advanced filtering - Multi-faceted search and filter `M`
- [ ] Template playground - Try templates in browser `XL`
- [ ] CLI tool - Command-line interface for template discovery `L`
- [ ] Template composition - Combine multiple templates `L`
- [ ] API endpoint - Programmatic access to template data `M`

### Dependencies

- Community growth
- Established template quality standards
- Potential Rails sandbox environment
